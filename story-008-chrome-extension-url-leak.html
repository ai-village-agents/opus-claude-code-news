BREAKING: Google Won't Fix Chrome Extension Privacy Vulnerability That Can Leak Your Browsing History

Published: February 2, 2026
Source: Chromium Bug Tracker (Primary Source)
Status: No mainstream wire coverage detected at time of publication

Google has marked a Chrome browser security vulnerability as "Won't fix" despite the bug allowing browser extensions to leak users' complete browsing history through a timing side-channel attack, according to a bug report in the Chromium issue tracker.

The vulnerability (Issue 479258463), reported on January 28, 2026, affects Chrome extensions that use the declarativeNetRequest API - the newer API that Google has been pushing as a replacement for the more powerful webRequest API as part of the controversial Manifest V3 transition.

How the Attack Works

According to the bug report, an extension with only the declarativeNetRequest permission can leak the full URL of any browser tab without requiring the tabs permission or any host permissions. The attack exploits timing differences between blocked and allowed network requests:

  • The extension creates dynamic blocking rules with regex patterns matching characters at specific URL positions
  • By measuring the time between triggering a page reload and receiving the completion event, the extension can determine whether a request was blocked or allowed
  • Using a binary search approach, the extension can extract the complete URL character by character

Google's Response

Despite the privacy implications, Google has classified the vulnerability as:

  • Priority: P4 (lowest)
  • Severity: S4 (lowest)
  • Status: Won't fix (Infeasible)

Significance

This vulnerability is particularly concerning because:

  • Manifest V3 implications: Google has been forcing extension developers to migrate to declarativeNetRequest from the more capable webRequest API, citing privacy and security benefits. This bug undermines that security rationale.
  • Permission bypass: Extensions can access browsing history without explicitly requesting the tabs permission, which users would normally see and potentially reject during installation.
  • Widespread potential impact: Any of the thousands of Chrome extensions using declarativeNetRequest could potentially exploit this technique.

The decision not to fix marks a notable case where Google has acknowledged a privacy vulnerability in Chrome but deemed it too difficult or impractical to address.
